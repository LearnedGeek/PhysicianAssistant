## 1. Project Overview

This SOW covers the Discovery phase of the DrOk platform — an AI-powered physician triage and patient communication system. The purpose of Discovery is to finalize requirements, resolve open regulatory questions, validate the technical architecture, and produce a complete specification sufficient to begin development.

---

## 2. Scope of Work

### 2.1 Deliverables

| # | Deliverable | Description |
|---|---|---|
| D1 | Functional Requirements Document | Complete RICEF-documented requirements mapped from clinical workflows |
| D2 | Technical Architecture Specification | Finalized system design, schema, API contracts, infrastructure plan |
| D3 | Authenticity Boundary Specification | Confabulation mitigation architecture: two-gate system, confidence thresholds, null response paths, emergency detection design |
| D4 | Emergency Protocol Definition | Physician-approved keyword list, critical lab value thresholds, escalation rules — signed off by Client |
| D5 | Data Flow & Privacy Compliance Map | End-to-end data flow diagram showing where data resides, who accesses it, and which legal framework governs each node |
| D6 | Regulatory Status Summary | Documented outcomes from Carlos Rojas review: DIGEMID classification, Ley 30421 applicability, APDP registration requirements |
| D7 | UI/UX Wireframes | Physician dashboard, admin portal, and patient conversation flow mockups |
| D8 | Development Phase SOW | Detailed SOW for Phases 2–5 with milestones, timelines, and payment schedule |

### 2.2 Activities

- Discovery call(s) with Client to finalize requirements
- Review and incorporation of Client's regulatory counsel findings
- Emergency keyword and critical lab threshold workshop with Client
- Architecture review incorporating ANI cross-project confabulation findings
- Wireframe review session with Client
- Preparation of development-phase SOW and pricing

### 2.3 Out of Scope

- Production code development (covered in subsequent SOW)
- Regulatory filings (Client responsibility per MSA Section 9)
- Legal document drafting (engagement letter, DPA — separate from this SOW)
- Marketing, sales, or business development activities
